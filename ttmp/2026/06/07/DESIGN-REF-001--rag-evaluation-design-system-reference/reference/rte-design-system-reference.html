Part I — Styling & Look-and-Feel

1. Design Tokens

Every visual value in this system is defined by a CSS custom property. Hard-coding a colour or spacing value anywhere else is not permitted.

TokenValueUsage
--mac-bg#FFFFFFPage / card backgrounds
--mac-text#000000Primary text
--mac-text-dim#666Secondary / muted text
--mac-surface-2#EEEAlternate rows, zebra stripes
--mac-accent#0000CCPrimary action / link colour
--mac-accent-2#CC0000Destructive actions
--mac-green#007722Success / OK status
--mac-amber#AA7700Warnings / pending

Part II — Widget System, Hierarchy & Interaction

4. Widget IR: The Core Data Structure

Every UI page is a JSON tree of widget nodes. There are three kinds of nodes:

KindFieldsDescription
textkind, textLeaf text
elementtag, attrs, childrenHTML passthrough
componenttype, props, childrenMapped to a React component by WidgetRenderer

6. Interaction: Actions

Widget nodes can carry an action property describing what happens on interaction.

KindBehaviour
navigateSPA push to a new page
serverPOST to server, receive new Widget IR
eventDispatch a custom DOM event
copyCopy text to clipboard

7. Semantic Recipes

Pre-composed patterns for common dashboard needs:

  • metrics() – grid of KPI tiles
  • actionToolbar() – header bar with buttons
  • masterDetailTable() – master-detail table layout
