2. Composition Patterns

Widgets are composed in a strict hierarchy. The tree must start with either AppShell or Stack as the root.

Pattern A: Dashboard

AppShell
 └── DashboardGrid (cols=2)
      ├── Panel (title="Metrics")
      │    └── Stack
      │         ├── StatusText (ok)
      │         └── Caption
      └── Panel (title="Query Results")
           └── DataTable

Pattern B: Master-Detail

Stack
 ├── DataTable (master)
 └── Panel (detail, conditionally shown)
      └── MetadataGrid

Pattern C: Settings Form

Panel (title="Settings")
  Stack
    ├── FormRow(label="Name", control=TextInput)
    ├── FormRow(label="Mode", control=SelectInput)
    └── FormRow(label="Advanced", control=CheckboxRow)

3. Component Details

3.1 Panel

PropTypeNotes
titleReactNodeHeader text. Omit for a border-only container
actionsReactNodeButton or toolbar on the header right
density'normal' | 'condensed'Controls internal padding
fillbooleanStretch to fill parent height

3.2 Stack

Vertical flexbox with optional gap. gap values: xs(2px), sm(4px), md(8px), lg(16px), xl(24px). align controls stretch, start, center, end.

3.3 Inline

Horizontal flexbox. Same gap tokens. justify supports start, between, end. Set wrap for wrapping.

3.4 DataTable

PropType
columns{ id, header, cell: (row) => ReactNode, align?, maxWidth? }[]
rowsArray of data objects
getRowKey(row) => string
onRowSelect(row) => void

3.5 StatusText

Shows a status indicator dot + label. Status values: ok, error, running, warning, pending, canceled. Optionally show icon (Unicode character) before the text.

3.6 MetadataGrid

Renders a <dl> of key-value pairs. Set density="compact" for tighter rows. Each item has optional copyValue that enables a copy-to-clipboard button.

3.7 Caption

Tones: muted, accent, success, warning, danger, inherit. Transforms: none, uppercase. With truncate for ellipsis on overflow.

3.8 Actions & Interactions

Every widget can carry an action object that describes what happens on click.

KindBehaviourTrigger
navigateSPA navigationURL change, no full reload
serverPOST /api/widget/actions/nameReloads page content from server
eventDispatches a CustomEventComponent communication
copyWrites to clipboardButton or detail view

5. Semantic Recipes

Pre-composed patterns exposed via the rag.recipes JavaScript object. These are convenience constructors that return Widget IR:

RecipeSignatureOutput
metricsrag.recipes.metrics({ items })DashboardGrid of StatusText/KPI tiles
actionToolbarrag.recipes.actionToolbar({ title, actions[] })Panel with row of Button widgets
masterDetailrag.recipes.masterDetailTable({ rows, columns, detail })Stack with DataTable + detail Panel

6. Action Flow Diagram

  User clicks
    ↓
  Action object (navigate | server | event | copy)
    ↓
  WidgetRenderer dispatches:
    server → POST /api/widget/actions/{name}
    navigate → router.push(href)
    event → dispatchEvent(new CustomEvent(…))
    copy → navigator.clipboard
